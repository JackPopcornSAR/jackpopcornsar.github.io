PARA — Why Rust

차량 소프트웨어에 왜 Rust인가?

차량 소프트웨어의 안전성과 품질 문제를 근본부터 해결합니다. 런타임 테스트 의존 대신 컴파일 타임에 메모리 오류를 제거합니다.

양산 후 결함 리스크 감소

Use-after-free, buffer overflow 등 메모리 버그가 컴파일 단계에서 차단됩니다. 리콜과 OTA 긴급 패치의 리스크를 낮춥니다.

멀티코어 HPC 안정성

Data race를 컴파일러가 방지합니다. 복잡한 멀티코어 ECU 환경에서도 동시성 버그 없이 안정적으로 동작하므로, 검증 비용과 기간을 절감합니다.

성능 타협 없는 안전성

Rust의 제로 코스트 추상화로 C/C++와 동등한 런타임 성능을 유지합니다. 안전성을 위해 하드웨어 스펙을 높일 필요가 없습니다.

기능 안전 인증에 유리

언어 수준의 결함 방지가 ISO 26262 ASIL 인증에서 증명해야 할 안전 케이스를 줄여줍니다. 더 적은 결함, 더 짧은 인증 기간을 기대할 수 있습니다.

PARA 차별점

PARA는 업계 유일한 Rust 네이티브 차량용 소프트웨어 플랫폼입니다. Container Manager를 통한 AI/ROS2 통합과 다양한 통신 프로토콜(SOME/IP, DDS, Zero-copy IPC, Zenoh)을 지원합니다.
